Old `brain_connectome_reservoir.py` (forward/tiling) → archive after migration. `reservoir.py` = random baseline (keep). `reservoir_ramiro.py` = **broken** (imports nonexistent `reservoirs/` pkg — which is exactly the layout we're building) → archive. The 5 `*.py` next to notebooks are derived nbconvert exports → delete/regenerate.
- **Traps:** best result (test **96.67%**) lives in an *old-engine* notebook (`humand_data_1015`) — reproduce before archiving. Many outputs are stale/interrupted/hand-pasted. Absolute Windows paths to private `folder_path_*` connectome dirs that don't ship. `transform()` docstring lies (returns `[T,N]`, not `[T,N+1]`).
- **Science gaps (paper-grade must fix):** train/test leakage from overlapping shuffled windows; a neuron sweep that secretly does nothing (`target_size` commented out); results computed on mock data (real connectomes now committable); **no null-model baseline (the #1 reviewer gap)**. Super-critical spectral radius (ρ>1) IS defensible for short driven windows (Yildiz 2012). **Symmetrization is NOT a gap** — connectomes are undirected (confirmed), so symmetric `W` is correct; the real-eigenvalue-only dynamics is a stated *limitation* (directed connectomes are unrecoverable via non-invasive imaging), not a bug.
- **Efficiency:** the `KeyboardInterrupt`s come from sweeps rebuilding identical reservoirs hundreds of times + an O(N³) eig in `reservoir.py` — all low-effort fixes. Bigger win: batch the per-timestep forward loop across windows (GEMM).
- **CL design anchor:** Cossu et al. 2021 — a frozen reservoir does NOT prevent forgetting (it moves to the readout); SLDA/conceptors are the strong readout-only CL methods; EWC fails in class-IL. So conceptors are the headline continual path.
