## 4. Distilled findings (detail lives in `CLEANUP_PLAN.md`)

- The duplication runs along two tracks forked many times. The 15 notebooks fall into 4 clusters: A=`handson_*` tutorials (~95 to 98% identical), B=`humand_data*` (~90%), C=`*slither_copy*` pipeline (~75 to 80%), D=3 unique standalones (`project1` continuous-time, `test_reservoirs`, `slither_io_test`).
- Engines. `brain_connectome_reservoir_v0_1.py` is canonical (fit/predict/ridge/washout/random-projection upscale). The old `brain_connectome_reservoir.py` (forward/tiling) goes to archive after migration. `reservoir.py` is the random baseline (keep). `reservoir_ramiro.py` is broken (it imports a nonexistent `reservoirs/` package, which is exactly the layout we are building) and goes to archive. The 5 `*.py` files next to notebooks are derived nbconvert exports, to delete and regenerate.
- Traps. The best result (test 96.67%) lives in an *old-engine* notebook (`humand_data_1015`); reproduce it before archiving. Many outputs are stale, interrupted, or hand-pasted. There are absolute Windows paths to private `folder_path_*` connectome dirs that don't ship. The `transform()` docstring lies (it returns `[T,N]`, not `[T,N+1]`).
- Science gaps (paper-grade must fix). Train/test leakage from overlapping shuffled windows; a neuron sweep that secretly does nothing (`target_size` commented out); results computed on mock data (real connectomes are now committable); and no null-model baseline (the #1 reviewer gap). A super-critical spectral radius (ρ>1) IS defensible for short driven windows (Yildiz 2012). Symmetrization is NOT a gap: connectomes are undirected (confirmed), so a symmetric `W` is correct; the real-eigenvalue-only dynamics is a stated *limitation* (directed connectomes are unrecoverable via non-invasive imaging), not a bug.
- Efficiency. The `KeyboardInterrupt`s come from sweeps rebuilding identical reservoirs hundreds of times, plus an O(N³) eig in `reservoir.py`, all low-effort fixes. The bigger win is to batch the per-timestep forward loop across windows (GEMM).
- CL design anchor: Cossu et al. 2021. A frozen reservoir does NOT prevent forgetting (it moves to the readout); SLDA and conceptors are the strong readout-only CL methods; EWC fails in class-IL. So conceptors are the main continual path.

## 5. Open questions, RESOLVED (user, 2026-06-25)

1. Erdős-Rényi: implement it for real. Build a genuine `ErdosRenyiReservoir` (a sparse random graph with connection probability `p`) in `reservoirs/random.py`, as a 4th connectivity option. This fulfils the intent of the broken `reservoir_ramiro.py` import (`from reservoirs.ErdosRenyi import ErdosRenyiReservoir`). The `handson_erdos_renyi` experiment becomes meaningful instead of a mislabeled re-run.
2. Private connectome data can be committed. Commit a representative subset of the real `folder_path_{83,129,234,463,1015}` GraphML connectomes so the paper's numbers are reproducible. This resolves regression risk R8 (results unrunnable on a fresh clone).
3. Connectomes are UNDIRECTED, so `symmetric=True` is correct and data-driven, NOT a methodological flaw. But an undirected connectome means a symmetric `W`, which means real-eigenvalue-only dynamics (no oscillatory or complex modes), and that IS a genuine limitation to state in the paper. The reason directed wasn't used, stated explicitly: non-invasive structural imaging (diffusion MRI / tractography) cannot resolve the directional trajectory of nerve fibres, so edge directionality is unrecoverable; directed connectomes are out of scope because of this measurement limitation plus the added complexity. See `CLEANUP_PLAN.md` §5.2.
4. CL scenario is DOMAIN-INCREMENTAL. The class/label set is fixed over time; only the input distribution shifts across sessions. The benchmark is domain-IL by design, so there is no scenario argument; construct it as `ContinualBenchmark(readout)`.
